# Yassir 2.0 — Predictive Compliance

**Don't just screen. Catch compliance drift before it hits your portfolio.**

Yassir now tracks the direction of a name's compliance ratios over time,
flags when a cached screen is too old to trust, and proposes halal
replacements for any holding that fails or starts drifting. You don't
have to ask — the agent runs these checks the moment they become useful.

```
> /monitor AAPL MSFT NVDA TSLA

Symbol  Verdict     Trend          Stale?  Suggested replacements
AAPL    COMPLIANT   stable          no     —
MSFT    COMPLIANT   debt rising     no     —  (drift watch)
NVDA    COMPLIANT   stable          yes    re-screen recommended
TSLA    COMPLIANT   debt declining  no     —
```

## What's new

### Predictive compliance, on by default

Three new capabilities turn drift into a first-class signal:

- **Trajectory** — see whether debt-to-market-cap, cash-and-securities,
  and non-compliant income are improving, stable, or deteriorating.
- **Staleness** — flag any cached screen that's too old, with the
  material SEC filings since the last screen and a re-screen
  recommendation.
- **Halal alternatives** — ranked, compliance-aware substitutes whenever
  a holding fails or starts drifting.

Yassir calls them automatically when they're useful: any COMPLIANT name
with marginal ratios gets a drift check, any NON_COMPLIANT candidate
gets alternatives, and stale screens trigger a refresh suggestion.

### New slash commands

- **`/monitor <SYMS>`** or **`/monitor watchlist:<name>`** — a single
  command produces a decision-grade table over an arbitrary symbol set
  or a named watchlist: `verdict · trend · stale? · suggested
  replacements`. Warns about expected token usage above five symbols.
- **`/trajectory <SYM>`** — direct drift check.
- **`/staleness <SYM>`** — direct cache-age check.
- **`/ideas <SYM>`** — now leans on compliance-ranked alternatives.
- **`/audit`** — follows up automatically on COMPLIANT names with
  marginal ratios; flags scholar-verified vs algorithmic-only
  methodologies.
- **`/usage`** — now shows 14-day burn, recent calls, and projected
  quota exhaustion at the current rate.

### Honest verdicts

- **ETFs.** When the backend provides v2 disposition
  (`compliant_with_purification`, `mostly_compliant`, or
  `non_compliant`) and scholar attestations, Yassir surfaces them
  with that nuance — no more boolean collapse for the v2 cohort.
  Legacy v1 ETF responses continue to render normally.
- **Stocks.** Confidence is labeled on every answer. ADR-driven
  currency mismatches abstain with `INSUFFICIENT_DATA` instead of
  forcing a wrong verdict.
- **Degraded upstreams.** When part of a check is unavailable (for
  example, a temporary EDGAR outage), Yassir says so once at the top of
  the answer instead of hiding it.

### Self-hoster quality-of-life

- Track usage, projected quota exhaustion, and recent call costs from
  inside the agent.
- Rotate or revoke API keys without leaving the chat.
- Automatic retries on rate-limit responses before any tool-level
  error.

### Evidence-quality discipline

The agent now:

- Labels every answer as *verified*, *algorithmic-only*, *abstained*,
  or *partial* before any verdict claim.
- Surfaces degraded sources once at the top of the answer when
  relevant.
- Never papers over an abstain with a confident claim.
- Treats unresolved or quota-blocked names as evidence absence, not
  compliance signals.
- Recommends a fresh re-screen when staleness fires, before any
  high-stakes decision.
- Distinguishes "no data" from "non-compliant" in summary language.

## Upgrading

```bash
git pull origin main
git checkout v2026.5.9
bun install
```

Drop-in replacement. Existing screens, watchlists, and prompts continue
to work; the new behaviors layer in transparently.

If you self-host with Docker:

```bash
docker compose build && docker compose up -d
```

## Token-cost note

`/monitor` runs trajectory + staleness for every symbol, with
alternatives only on flagged names — roughly 2–3 calls per symbol. The
command warns up front when you ask it to run over more than five
names.
